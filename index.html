Cybersecurity Fundamentals: Module 1

Understanding the Cybersecurity Landscape

Introduction to Cybersecurity

Cybersecurity is the practice of protecting systems, networks, and programs from digital attacks. These attacks are usually aimed at accessing, changing, or destroying sensitive information; extorting money from users; or interrupting normal business processes.

Exercise: Identify Common Threats

Match these common cyber threats with their descriptions:

  1. Phishing
  2. Malware
  3. DDoS Attack
  4. SQL Injection

Descriptions:

  • A. Malicious software designed to harm computer systems
  • B. Flooding a server with traffic to make it unavailable
  • C. Tricking users into revealing sensitive information
  • D. Injecting malicious code into database queries

Key Cybersecurity Concepts

Understanding these fundamental concepts is crucial for any cybersecurity professional:

  • Confidentiality: Ensuring that information is accessible only to those authorized to have access
  • Integrity: Safeguarding the accuracy and completeness of information and processing methods
  • Availability: Ensuring that authorized users have access to information and associated assets when required

# Example: Basic security check command
whoami                    # Check current user
netstat -tuln             # Check open ports
ps aux                    # View running processes

Network Security: Module 1

Network Fundamentals and Security Principles

Understanding Network Protocols

Network protocols are established rules that determine how data is transmitted between devices. Key protocols include:

  • TCP/IP: The fundamental communication protocol of the internet
  • HTTP/HTTPS: Protocols for web browsing (HTTPS adds encryption)
  • DNS: Translates domain names to IP addresses
  • SSH: Secure protocol for remote access to systems

Exercise: Basic Network Scanning

Use the following Nmap command to scan a network:


nmap -sS 192.168.1.0/24

This command performs a SYN scan on the 192.168.1.0/24 network. Try it in your lab environment and document:

  1. How many hosts are online
  2. What services are running on each host
  3. Which ports are open
