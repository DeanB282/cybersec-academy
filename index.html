Module 1: Cybersecurity Fundamentals

Understanding the Cybersecurity Landscape

Introduction to Cybersecurity

Cybersecurity is the practice of protecting systems, networks, and programs from digital attacks. These attacks are usually aimed at accessing, changing, or destroying sensitive information; extorting money from users; or interrupting normal business processes.

Key Cybersecurity Concepts

Understanding these fundamental concepts is crucial for any cybersecurity professional:

  • Confidentiality: Ensuring that information is accessible only to those authorized to have access
  • Integrity: Safeguarding the accuracy and completeness of information and processing methods
  • Availability: Ensuring that authorized users have access to information and associated assets when required

Module 2: Network Security Basics

Understanding Network Protocols and Security

Network Protocols and Security

Network protocols are established rules that determine how data is transmitted between devices. Understanding these protocols is essential for securing network communications.

Exercise: Network Scanning with Nmap

Use the following Nmap commands to practice network scanning:


# Basic network scan
nmap -sS 192.168.1.0/24

# Service version detection
nmap -sV 192.168.1.1

# OS detection
nmap -O 192.168.1.1

# Aggressive scan
nmap -A 192.168.1.1

Practice these commands in the Interactive Lab on the homepage and answer:

  1. What is the difference between -sS and -sV scans?
  2. Which flag would you use to detect the operating system?
  3. What does the -A flag do in Nmap?
Hint: The -sS flag performs a SYN scan (stealth), while -sV detects service versions.

Common Network Attacks

Understanding common network attacks helps in developing effective defenses:

  • Man-in-the-Middle (MitM): Intercepting communications between two parties
  • DNS Spoofing: Corrupting DNS cache to redirect traffic
  • ARP Poisoning: Manipulating ARP tables to redirect traffic
  • Port Scanning: Identifying open ports and services

CI/CD Pipeline Security Challenge

Identify and fix security vulnerabilities in a Jenkins pipeline

Challenge Instructions

You've been given a Jenkins pipeline configuration that contains several security vulnerabilities. Your task is to identify and fix these vulnerabilities to make the pipeline secure.

  • Time: 45 minutes
  • Difficulty: Intermediate
  • Tools: Jenkins, Docker, Snyk

Your Tasks

  1. Identify hardcoded credentials in the pipeline
  2. Fix insecure Docker image usage
  3. Implement proper secret management
  4. Add security scanning steps
  5. Ensure proper access controls

Vulnerable Jenkinsfile

Review this Jenkinsfile and identify the security issues:


pipeline {
    agent any
    environment {
        AWS_ACCESS_KEY = 'AKIAIOSFODNN7EXAMPLE'
        AWS_SECRET_KEY = 'wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY'
        DOCKER_PASSWORD = 'mypassword123'
    }
    stages {
        stage('Build') {
            steps {
                sh 'docker build -t myapp:latest .'
                sh 'docker run --privileged myapp:latest'
            }
        }
        stage('Test') {
            steps {
                sh 'npm test'
            }
        }
        stage('Deploy') {
            steps {
                sh 'docker push myapp:latest'
                sh 'curl -X POST http://prod-server/deploy -d "image=myapp:latest"'
            }
        }
    }
}

Security Issues to Look For

  • Hardcoded credentials in environment variables
  • Use of privileged Docker containers
  • Lack of image scanning
  • Insecure deployment methods
  • Missing access controls

Fix the Jenkinsfile

Rewrite the Jenkinsfile to address the security issues:

Secure_Jenkinsfile
pipeline { agent any environment { // Use Jenkins credentials store instead of hardcoded values } stages { stage('Build') { steps { script { // Add security scanning here } sh 'docker build -t myapp:latest .' // Remove privileged flag and use specific capabilities } } stage('Security Scan') { steps { // Add container security scanning } } stage('Test') { steps { sh 'npm test' } } stage('Deploy') { steps { // Use secure deployment methods } } } }

Adversarial AI Attack Challenge

Create and defend against adversarial machine learning attacks

Challenge Instructions

In this challenge, you'll work with an image classification model to understand adversarial attacks and implement defenses.

  • Time: 60 minutes
  • Difficulty: Advanced
  • Tools: TensorFlow, PyTorch, Adversarial Robustness Toolbox

Part 1: Create Adversarial Examples

Use this Python code to create adversarial examples that fool the classifier:


import tensorflow as tf
import numpy as np
from art.attacks.evasion import FastGradientMethod
from art.classifiers import TensorFlowV2Classifier

# Load pre-trained model
model = tf.keras.applications.ResNet50(weights='imagenet')

# Create classifier wrapper
classifier = TensorFlowV2Classifier(
    model=model,
    nb_classes=1000,
    input_shape=(224, 224, 3)
)

# Create adversarial examples
attack = FastGradientMethod(estimator=classifier, eps=0.3)
adversarial_images = attack.generate(x_original)

# Test attack success
predictions_original = model.predict(x_original)
predictions_adversarial = model.predict(adversarial_images)

Task: Modify the epsilon (eps) value and observe how it affects the attack success rate.

Part 2: Implement Defense

Add adversarial training to make the model more robust:


# Adversarial training function
def adversarial_training(model, x_train, y_train, epochs=10):
    for epoch in range(epochs):
        # Generate adversarial examples
        adversarial_x = attack.generate(x_train)
        
        # Combine original and adversarial data
        combined_x = np.concatenate([x_train, adversarial_x])
        combined_y = np.concatenate([y_train, y_train])
        
        # Train on combined dataset
        model.fit(combined_x, combined_y, epochs=1, verbose=0)
        
    return model
